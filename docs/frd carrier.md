## 1. Introduction

### 1.1 Purpose

The **Quoting Engine** is a specialized, stateless microservice responsible for the real-time calculation of logistics shipping rates. It aggregates raw costs from external carriers (simulated), normalizes data, applies business profit margins, and returns a finalized sell price to the Core Monolith.

### 1.2 Scope

**In Scope:**

* Receiving Quote Requests via gRPC.
* Validating Locations (UN/LOCODE).
* Fetching raw rates from the Carrier Simulator (REST) using the **corrected v1.1 Data Contract**.
* Currency Conversion & Normalization.
* **Dynamic Surcharge Aggregation** (handling variable carrier fees).
* Application of Volumetric Weight logic.
* Application of Dynamic Profit Margins.
* Caching of results in Redis.

**Out of Scope:**

* User Authentication.
* Booking Execution.
* Payment Processing.

## 2. System Interfaces

### 2.1 Input Interface (Internal API)

The service MUST expose a **gRPC Server** listening on port `50051`.

* **Service Definition:** `QuotingService`
* **Method:** `GetQuotes(QuoteRequest)`
* **Payload (Protobuf):**
* `transaction_id` (UUID): For tracing.
* `origin_locode` (String): e.g., "CNSHA".
* `destination_locode` (String): e.g., "ESBCN".
* `ready_date` (String): ISO8601 Date (YYYY-MM-DD).
* `cargo_details` (Object):
* `equipment_type` (Enum): **20FT, 40FT, 40HC, 40RF, LCL**. *(Updated to include Reefer)*
* `total_weight_kg` (Double): Floating point precision.
* `total_volume_cbm` (Double): Floating point precision.
* `commodity_type` (Enum): GENERAL, HAZMAT, PERISHABLE.


* `incoterms` (Enum): FOB, CIF, EXW.
* `currency_requested` (String): EUR, USD.



### 2.2 Output Interface (Internal API)

The service MUST return a `QuoteResponse` Protobuf message.

* **Payload:**
* `quotes` (List):
* `quote_id` (UUID): Unique ID generated by the Engine.
* `carrier_name` (String).
* `total_price` (Decimal/String).
* `currency` (String).
* `transit_time_days` (Int).
* `valid_until` (Timestamp).
* `breakdown` (Object): Base Freight, Surcharges, Fees.





### 2.3 External Interface (Carrier Integration)

The service MUST act as a **REST Client** to query the Carrier Simulator.

* **Protocol:** HTTP/1.1 (Keep-Alive).
* **Timeout:** Strict **2000ms** hard timeout per request.
* **Circuit Breaker:** Open state after 5 consecutive failures.
* **Data Contract:** See Section 7.2 (Revised JSON).

## 3. Detailed Functional Requirements

### 3.1 Input Validation & Master Data

**FR-QUOTE-001: Location Validation**
The system MUST validate `origin_locode` and `destination_locode` against the local Redis Cache (`REF_DATA:LOCATIONS`).

* **Logic:** If a code does not exist in Redis, return `INVALID_ARGUMENT`.

**FR-QUOTE-002: Cargo Constraints**
The system MUST validate physical constraints of the cargo.

* **Logic:**
* If `equipment_type == '20FT'` AND `weight > 28,000kg` -> Error "Max Payload Exceeded".
* If `equipment_type == '40FT'` OR `40HC` OR `40RF` AND `weight > 30,000kg` -> Error "Max Payload Exceeded".



### 3.2 Aggregation Logic (The Fan-Out)

**FR-QUOTE-003: Concurrent Carrier Querying**
The system MUST spawn a separate Go Routine for each configured Carrier/Mock endpoint.

* **Performance:** Queries must execute in parallel using `sync.WaitGroup`.

**FR-QUOTE-004: Carrier Selection Strategy**

* **MVP Logic:** Query "Carrier Simulator" for all routes.

**FR-QUOTE-005: Resiliency & Timeouts**
The system MUST enforce a "Global Timeout Budget".

* **Logic:** If gRPC Context deadline is 2s, internal HTTP Client timeout = 1.95s.

### 3.3 The Calculation Engine (Core Logic)

**FR-QUOTE-006: Currency Normalization**
The system MUST convert carrier rates to the `currency_requested`.

* **Process:** Fetch `RATES:EXCHANGE:{CARRIER_CUR}:{USER_CUR}` from Redis.
* **Calc:** `CarrierPrice * ExchangeRate`.

**FR-QUOTE-007: Dynamic Cost Aggregation (Revised)**
The system MUST aggregate the Base Rate and **ALL** surcharges returned by the carrier.

* **Old Logic:** Sum specific codes (BAF, CAF).
* **New Logic (Fix):** Iterate through the `surcharges` array in the JSON response. Sum the `amount` field of **every** object found, regardless of the `code`.
* **Formula:** `TotalCost = BaseFreight + SUM(surcharges[i].amount)`.

**FR-QUOTE-008: Volumetric Weight (Chargeable Weight)**
For `LCL` requests only:

* **Formula:** `VolumetricWeight = VolumeCBM * 1000`.
* **Logic:** `ChargeableWeight = MAX(ActualWeightKg, VolumetricWeight)`.
* **Usage:** Apply Carrier LCL Rate (Price per Unit) to `ChargeableWeight`.

**FR-QUOTE-009: Overweight Surcharge (OWS) (Revised)**
The system MUST apply an internal surcharge if the carrier did not, but the weight exceeds the **standardized threshold**.

* **Threshold (Fixed):** **21,500 kg**. (Previously 21,000 kg).
* **Logic:**
1. Check if `total_weight_kg > 21,500`.
2. Check if Carrier Response `surcharges` array contains code `OWS`.
3. IF (Weight > 21,500) AND (Carrier has NO OWS) -> **Add Internal Fee $150.00**.



### 3.4 Profit Margin Application

**FR-QUOTE-010: Margin Rules**
Apply margins on top of `TotalCost`.

* **Step 1:** `SellPrice = TotalCost * (1 + Margin_Percentage)` (e.g., 1.15).
* **Step 2:** `SellPrice = SellPrice + Fixed_Handling_Fee` (e.g., 25.00).

**FR-QUOTE-011: Smart Tiering**

* If `incoterm == 'EXW'`, add extra `+ $50.00` handling fee.

### 3.5 Output Generation

**FR-QUOTE-012: Response Sorting**

* **Primary:** Total Price (Ascending).
* **Secondary:** Transit Time (Ascending).

**FR-QUOTE-013: ID Generation**
Generate deterministic UUID based on hash of inputs + final price.

## 4. Data Requirements

### 4.1 Caching Strategy (Redis)

**FR-DATA-001: Request Hashing**
Generate SHA-256 hash of (`origin` + `dest` + `weight` + `equipment` + `ready_date`) as Cache Key.

**FR-DATA-002: Cache Lifecycle**

* **TTL:** 10 minutes.
* **Read:** Check Redis first. If Hit -> Return immediately.

### 4.2 Redis Schema

```
REF:PORTS:{UNLOCODE}         -> Hash { "name": "Shanghai", "zone": "Asia" }
RATES:EXCHANGE:{FROM}:{TO}   -> String "0.92"
CACHE:QUOTES:{SHA256_HASH}   -> String (JSON Blob)

```

## 5. Non-Functional Requirements (NFRs)

### 5.1 Performance

* **NFR-PERF-001:** CPU calculation time **< 10ms**.
* **NFR-PERF-002:** Memory footprint < 2MB per request.

### 5.2 Error Handling

* **NFR-ERR-001:** If Carrier Mock returns `5xx`, log error and exclude that carrier. Do not fail request unless list is empty.
* **NFR-ERR-002:** If Carrier Mock returns `429`, trigger Circuit Breaker.

### 5.3 Logging

* **NFR-LOG-001:** Structured JSON logs.
* **NFR-LOG-002:** Include `transaction_id`.
* **NFR-LOG-003:** Log `price_delta` (Cost vs Sell).

---

## 6. Edge Cases & Constraints

* **Zero Results:** Return empty list, status OK.
* **Negative Margins:** Safeguard: `SellPrice` must never be less than `TotalCost`.
* **Past Dates:** If `ready_date` is in the past, return `INVALID_ARGUMENT`.

```json
{
  "origin": "CNSHA",
  "destination": "ESBCN",
  "departureDate": "2026-02-15",  // MAPPED FROM ready_date

  "weight": 21500.50,             // NUMBER (Float), NOT Integer

```

```json
{
  "carrierId": "MAEU",
  "baseRate": 2500.00,
  "currency": "USD",
  "surcharges": [
    { "code": "BAF", "amount": 150.00 },
    { "code": "PSS", "amount": 500.00 }, // Quoting Engine MUST sum this dynamically
    { "code": "OWS", "amount": 200.00 }
  ],
  "validUntil": "2026-03-01"
}

```
